제품 피드백

강점 · 개선요청
강점
레코딩·재현 속도 빠름 → 대량 테스트에 유리
에너지 솔루션 1.0 → 2.0 적용 과정에서 3자 테스트(지선민 팀장 측)를 통해 확인된 검증된 강점
개선 요청
엔드투엔드 프로세스 시나리오 테스트 부족
T-code 단위가 아닌 업무 흐름 전체를 하나의 시나리오로 테스트하는 관점이 부족했음. 유지보수·결과 가시성도 함께 강화 요청
개선 요청
백그라운드 실행 시 증적(Evidence) 부족
일본·미국 고객은 테스트 결과 제출 요건이 높음 → 백그라운드 실행 중에도 단계별 화면 캡처·입력값 기록 확보 기능 요청. 현재 개선 완료
기능 요청
비 SAP 시스템(OWMS 등) 연계 테스트
초기 WebGUI 중심 레코딩으로는 타 웹 시스템 연계에 한계. CJ 제일제당·에너지 솔루션 등 현장에서 OWMS 등 유관 시스템까지 하나의 흐름으로 테스트 요구
기능 요청
T-code 중복 레코딩 해소 → 모듈화 재사용
VA01 같은 공통 T-code를 시나리오마다 다시 레코딩하는 비효율 해소 요청. 유닛 단일 수정이 참조 시나리오 전체에 자동 반영되는 구조로 개선 완료
긍정 피드백
Tricentis 경험자 기준에서도 사용성 우수
드래그앤드랍 방식의 직관성, AI 기반 자연어 시나리오 수정 편의성에 대해 긍정적 반응. 사용 대상을 컨설턴트·PI·고객층까지 확대 가능성 인정

세일즈 · 발표 피드백

전무 지시 포함
전무 지시
경영진 발표는 기대효과 중심으로 재구성
맵핑 상세 설명보다 핵심 포인트 3개 중심으로 압축 — ① 맵핑 불필요 ② 캡처 가능 ③ 시나리오 기반 흐름 테스트. 비용 절감·품질 향상 정량/정성 지표로 임팩트 강조
결정
경쟁 솔루션(Tricentis 등) 비교 언급 지양
자사 솔루션 가치 중심 설명 원칙 확정. 과도한 비교·최고식 주장 지양하고 가능한 기능만 명확히 설명
개선 요청
데모 영상 가시성 확보 (큰 화면 요소)
상위 임원 참석 시 영상 중심 메시지 전달 권장. 화면 요소를 크게 구성해 비전문가 경영진도 직관적으로 이해할 수 있도록 재촬영 검토 필요
개선 요청
의사결정권자 관심사 맞춤 시나리오 데모
일반적인 기능 시연이 아닌 고객 의사결정권자의 실제 관심 업무 흐름 중심으로 데모 재구성 필요 (일부 레코딩 재촬영 검토 요청)
